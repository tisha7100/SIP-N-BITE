DATA DICTIONARY

Database Name: food_snb

Main System Tables Analysis

users:

It stores registration and authentication details of the customers.

No. Field Name Data Type Size Constrain Description
1 id BigInt 20 Primary Key, AI Unique user identifier
2 name Varchar 255 Not Null User's full name
3 email Varchar 191 Unique, Not Null Email address for login
4 password Varchar 255 Not Null Bcrypt hashed password
5 status Enum - 'active','blocked' Account status
6 registration_time Timestamp - Nullable Time of user signup
categories:

It stores different food categories for menu classification.

No. Field Name Data Type Size Constrain Description
1 id BigInt 20 Primary Key, AI Unique category ID
2 name Varchar 255 Not Null Category name (e.g. Pizza)
3 image Varchar 255 Nullable Path to category icon/image
tables_layout:

It stores the physical layout and status of restaurant tables.

No. Field Name Data Type Size Constrain Description
1 id BigInt 20 Primary Key, AI Unique table ID
2 table_number Varchar 255 Unique, Not Null Display number of the table
3 capacity Unsigned Int - Not Null Number of seats available
4 status Enum - 'available','booked' Real-time booking status
5 is_active Boolean - Default true Active status in layout
table_bookings:

It stores reservation requests made by users for specific tables.

No. Field Name Data Type Size Constrain Description
1 id BigInt 20 Primary Key, AI Booking reference ID
2 user_id BigInt 20 Foreign Key Reference to users table
3 table_id BigInt 20 Foreign Key Reference to layout table
4 booking_date Date - Not Null Scheduled date
5 status Enum - Pending/Accepted... Current status of request
cart:

It stores temporary items added by users before checkout.

No. Field Name Data Type Size Constrain Description
1 id BigInt 20 Primary Key, AI Cart item unique ID
2 user_id BigInt 20 Foreign Key The owner of the cart
3 food_id BigInt 20 Foreign Key Item added to cart
4 quantity Unsigned Int - Not Null Number of items
orders:

It stores finalized order transactions and delivery details.

No. Field Name Data Type Size Constrain Description
1 id BigInt 20 Primary Key, AI Order reference number
2 user_id BigInt 20 Foreign Key The ordering customer
3 total_amount Decimal 10,2 Not Null Base amount
4 final_amount Decimal 10,2 Not Null Amount after tax/charges
5 status Enum - Default prep Lifecycle of the order
bills:

It stores generated invoices for completed orders.

No. Field Name Data Type Size Constrain Description
1 id BigInt 20 Primary Key, AI Invoice entry ID
2 order_id BigInt 20 Foreign Key Reference to parent order
3 bill_number Varchar 255 Unique Public bill number
4 final_amount Decimal 10,2 Not Null Final payable amount
5 payment_method Varchar 255 Not Null Mode of transaction
